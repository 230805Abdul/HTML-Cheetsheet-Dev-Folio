// Build It Yourself

Project: DevFolio — A Developer Portfolio

Build a complete developer portfolio website step-by-step. Each task introduces a new set of HTML tags. Complete all 10 tasks and you'll have a fully structured, semantic HTML website using almost every important HTML tag.

You'll build DevFolio — a personal portfolio for a fictional developer named Alex Rivera. The project is split into 10 tasks, each targeting specific HTML tags. Your job is to write the HTML for each task inside a single index.html file. No CSS or JavaScript required — pure HTML only!

HTML Only No CSS Required
Overall Progress 0 / 10 tasks completed
01 Document Skeleton
<!DOCTYPE><html><head> <meta><title><body>
To Do ▶

Every HTML file must start with a proper document structure. Your first task is to create the complete document skeleton for your portfolio. This includes the doctype declaration, root element, and all essential head metadata.

// Starter Template — complete the missing parts
<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8" /> <meta name="viewport" content="width=device-width, initial-scale=1.0" /> <meta name="description" content="___YOUR DESCRIPTION___" /> <meta name="author" content="Alex Rivera" /> <title>Alex Rivera — Full Stack Developer</title> </head> <body> <!-- Your content goes here --> </body> </html>
  • Create index.html and add <!DOCTYPE html>
  • Add <html lang="en"> as root element
  • Add <head> with charset, viewport, and description <meta> tags
  • Add <title> with your portfolio title
  • Add <body> as the content container
02 Navigation Bar
<nav><ul><li><a>
To Do ▶

Create the site navigation bar at the top of your portfolio. Use semantic HTML to build a proper nav element with an unordered list of anchor links pointing to each section of the page using ID anchors (e.g. #about).

// Your task
<nav> <ul> <li><a href="#hero">Home</a></li> <li><a href="#about">About</a></li> <!-- Add: Skills, Projects, Contact --> </ul> </nav>
  • Use <nav> as the semantic wrapper
  • Use <ul> → <li> for the list of links
  • Add 5 anchor links: Home, About, Skills, Projects, Contact
  • Each <a> should use href="#section-id" for in-page navigation
03 Hero Section
<header><h1>–<h3><p> <strong><em><br><hr>
To Do ▶

Build the hero / intro section at the top of the page. This should introduce Alex Rivera with a large heading, subtitle, short bio paragraph, and a horizontal divider below it.

// Structure to build
<header id="hero"> <h1>Hi, I'm <strong>Alex Rivera</strong></h1> <h2>Full Stack Developer & Open Source Enthusiast</h2> <p> I build <em>fast, accessible</em>, and beautiful web apps.<br> Based in Karachi, Pakistan. Available for freelance. </p> <hr> </header>
  • Wrap everything in <header id="hero">
  • Use <h1> for the name, <strong> to bold it
  • Use <h2> for the job title
  • Use <p> with <em> for emphasis and <br> for a line break
  • End with <hr> as a visual divider
04 About Section
<section><article><aside> <figure><figcaption><img><dl>
To Do ▶

Create the About Me section with a profile image, a description article, and a sidebar (aside) with quick facts using a description list. Use <figure> to wrap the image with a caption.

// Structure
<section id="about"> <h2>About Me</h2> <figure> <img src="profile.jpg" alt="Alex Rivera" width="200"> <figcaption>Alex Rivera — Karachi, 2024</figcaption> </figure> <article> <p>I'm a passionate developer with 5+ years of experience...</p> </article> <aside> <dl> <dt>Location</dt> <dd>Karachi, PK</dd> <dt>Experience</dt> <dd>5 Years</dd> <!-- Add: Freelance, Languages, etc. --> </dl> </aside> </section>
  • Wrap in <section id="about">
  • Add <figure> + <img> + <figcaption> for profile picture
  • Use <article> for the bio text
  • Use <aside> for sidebar quick facts
  • Use <dl> + <dt> + <dd> for the fact list (at least 4 items)
05 Skills Table
<table><thead><tbody> <tr><th><td><caption>
To Do ▶

Build a Skills Table displaying Alex's technical skills, proficiency level, and years of experience. Use proper table structure with a caption, header row, and body rows.

// Table Structure
<section id="skills"> <h2>My Skills</h2> <table> <caption>Technical Skills Overview</caption> <thead> <tr> <th>Skill</th> <th>Level</th> <th>Years</th> </tr> </thead> <tbody> <tr> <td>HTML & CSS</td> <td>Expert</td> <td>5</td> </tr> <!-- Add at least 4 more skills --> </tbody> </table> </section>
  • Use <table> with a <caption>
  • Add <thead> with a <tr> of <th> headers (Skill, Level, Years)
  • Add <tbody> with at least 5 rows using <tr> + <td>
  • Add a <tfoot> row summarizing total skills
06 Projects Section
<main><details><summary> <blockquote><code><pre><abbr>
To Do ▶

Build the Projects Section inside a <main> wrapper. Use expandable <details> cards for each project, with a testimonial quote and a code snippet preview.

// Project Card with details
<main> <section id="projects"> <h2>Projects</h2> <details> <summary>Project 1: Task Manager App</summary> <p>A <abbr title="Progressive Web App">PWA</abbr> task manager...</p> <pre><code> const task = { id: 1, title: "Buy milk", done: false }; </code></pre> <blockquote cite="https://client.com"> "This app saved our team 3 hours a day!" <cite>— Sarah K., Project Manager</cite> </blockquote> </details> <!-- Add 2 more project cards --> </section> </main>
  • Wrap in <main> as the primary content region
  • Use <details> + <summary> for 3 expandable project cards
  • Each card has a <p> description and a <pre><code> snippet
  • Add <blockquote> + <cite> for a client testimonial
  • Use <abbr title="..."> for at least one technical abbreviation
07 Contact Form
<form><input><label> <textarea><select><fieldset><button>
To Do ▶

Build a fully structured Contact Form using HTML form elements. Group the fields with <fieldset>, use proper <label> elements, and include various input types.

// Form Structure
<section id="contact"> <form action="#" method="post"> <fieldset> <legend>Personal Info</legend> <label for="name">Full Name</label> <input type="text" id="name" name="name" required> <!-- Add: email input, phone input --> </fieldset> <fieldset> <legend>Project Details</legend> <label for="type">Project Type</label> <select id="type" name="type"> <option value="web">Web App</option> <option value="mobile">Mobile App</option> </select> <label for="message">Message</label> <textarea id="message" name="message" rows="5"></textarea> </fieldset> <button type="submit">Send Message</button> <button type="reset">Clear</button> </form> </section>
  • Use <form method="post"> as container
  • Group with 2 <fieldset> + <legend> blocks
  • Add inputs: text, email, tel, number, date, checkbox, radio
  • Add <select> dropdown with <option> items
  • Add <textarea> for the message
  • Add submit and reset <button> tags
  • Every input must have a matching <label for="id">
08 Media Section
<video><audio><source> <picture><iframe>
To Do ▶

Add a Media Section to showcase a demo video, background music, a responsive image, and an embedded YouTube video using an iframe.

// Media embeds
<section id="media"> <h2>Demo Reel</h2> <!-- HTML5 Video --> <video controls width="640" poster="thumb.jpg"> <source src="demo.mp4" type="video/mp4"> Your browser does not support video. </video> <!-- HTML5 Audio --> <audio controls> <source src="intro.mp3" type="audio/mpeg"> </audio> <!-- Responsive Picture --> <picture> <source media="(max-width:600px)" srcset="small.jpg"> <img src="large.jpg" alt="Portfolio Preview"> </picture> <!-- YouTube Embed --> <iframe src="https://www.youtube.com/embed/VIDEO_ID" width="560" height="315" allowfullscreen></iframe> </section>
  • Add <video controls> with a <source> tag
  • Add <audio controls> with a <source> tag
  • Add <picture> with two <source> + fallback <img>
  • Embed a YouTube video using <iframe>

  • Wrap copyright in <small>
